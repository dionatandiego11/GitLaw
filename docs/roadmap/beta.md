# Beta

## Objetivo

Transformar o alpha em arquitetura de aplicacao mais robusta e auditavel.

## Prioridades

1. separar `domain`, `application` e `infrastructure`
2. introduzir autenticacao por assinatura
3. substituir JSON store por persistencia com auditoria
4. formalizar maquina de estados
5. modelar parecer e admissibilidade como entidades

## Entregas esperadas

- API menos acoplada ao Express
- testes de dominio mais fortes
- eventos de dominio explicitos
- fronteira clara entre off-chain e on-chain

## Criterio de sucesso

Ter um sistema tecnicamente solido como plataforma institucional, mesmo antes de uma integracao protocolar total.
